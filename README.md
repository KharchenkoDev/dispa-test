## Описание реализации

Основное задание реализовано в `app/src/Controller/InnController.php`. Получаем запрос, валидируем, запрашиваем данные в DaData и сохраняем в базу. Учтены ситуации, когда от DaData может прийти ошибка или неполные данные. Также возможность конкурентных запросов (если одновременно произошло два запроса по одинаковому ИНН).

После получения данных из DaData они записываются в базу данных и считаются валидными в течение часа. Кэширования не было в техзадании, но такое решение само напрашивается в подобной схеме. Можно легко изменить время жизни данных или отключить этот функционал при ненадобности.

Также для удобства реализована простая html страница (http://localhost:8080), где можно увидеть сохранённые на данный момент данные.

## Установка и запуск

- `git clone https://github.com/KharchenkoDev/dispa-test.git`
- `cd dispa-test`
- `make init`
- Прописать токен DaData в `app/.env`: `DADATA_TOKEN=ваш_токен`

Токен можно получить на [dadata.ru](https://dadata.ru) после регистрации.

## API

`GET /api/inn/{inn}` — поиск организации по ИНН (10 или 12 цифр).

```json
{
    "inn": "7707083893",
    "name": "ПАО Сбербанк",
    "is_active": true,
    "okved": "64.19",
    "okved_name": "Деятельность по предоставлению прочих видов кредита"
}
```
